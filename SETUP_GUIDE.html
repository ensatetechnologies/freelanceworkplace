1. Prerequisites

  • Python 3.11+ (tested on 3.13 too) — python.org
  • Git — git-scm.com
  • Either MySQL 8.0 (recommended, matches production) or the built-in SQLite for quick local use
  • A C/C++ build toolchain if you install mysqlclient from source (Windows: “Microsoft C++ Build Tools”; Debian/Ubuntu: sudo apt install build-essential python3-dev default-libmysqlclient-dev pkg-config)

2. Clone the repository

git clone https://github.com/ensatetechnologies/freelanceworkplace.git
cd freelanceworkplace

3. Create a virtual environment

Windows (PowerShell):

python -m venv venv
venv\Scripts\Activate.ps1

If PowerShell blocks activation: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned.

macOS / Linux:

python3 -m venv venv
source venv/bin/activate

4. Install dependencies

pip install --upgrade pip
pip install -r requirements.txt
Windows hint: if mysqlclient fails to build, install a prebuilt wheel: pip install mysqlclient usually works on 3.11/3.12, or download the matching wheel from Gohlke’s archive. Alternatively use SQLite (see step 6 below) and skip MySQL entirely.

5. Configure environment variables

# Windows PowerShell
Copy-Item env.example .env

# macOS / Linux
cp env.example .env

Open .env and set values for your machine:

DEBUG=1
SECRET_KEY=change-me-to-a-long-random-string
ALLOWED_HOSTS=localhost,127.0.0.1

DB_NAME=freelance_marketplace
DB_USER=root
DB_PASSWORD=your-mysql-root-password
DB_HOST=localhost
DB_PORT=3306

EMAIL_BACKEND=django.core.mail.backends.console.EmailBackend
SITE_URL=http://localhost:8000

6. Create the database

Option A — MySQL (recommended)

Log in to MySQL and create the database with proper collation:

CREATE DATABASE freelance_marketplace
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

Option B — SQLite (zero-setup)

Edit config/settings/development.py and replace the DATABASES block with:

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.sqlite3',
        'NAME': BASE_DIR / 'db.sqlite3',
    }
}

7. Run migrations & load fixtures

python manage.py makemigrations
python manage.py migrate
python manage.py loaddata fixtures/initial_data.json

8. Create / verify a superuser

If the fixture does not ship with an admin on the target machine, create one:

python manage.py createsuperuser

Or create / reset one non-interactively to match the provided credentials:

python manage.py shell -c "from django.contrib.auth import get_user_model; U=get_user_model(); u,_=U.objects.get_or_create(username='admin', defaults={'email':'admin@example.com','role':'admin'}); u.is_superuser=True; u.is_staff=True; u.role='admin'; u.set_password('admin123'); u.save()"

9. Start the development server

python manage.py runserver

Then open:

  • Main site → http://localhost:8000
  • Django admin → http://localhost:8000/admin
  • Admin dashboard → http://localhost:8000/dashboard

10. Demo login credentials

11. Troubleshooting

Port 8000 already in use

# Windows: kill all Django runservers
Get-CimInstance Win32_Process -Filter "Name='python.exe' AND CommandLine LIKE '%runserver%'" | ForEach-Object { Stop-Process -Id $_.ProcessId -Force }

# macOS / Linux
lsof -ti:8000 | xargs kill -9

mysqlclient fails to install

  • Install the OS-level MySQL client dev headers (see Prerequisites).
  • Or temporarily switch to SQLite (step 6 option B).

Static files look unstyled

Hard-refresh with Ctrl+F5. In production runs, collect static files: python manage.py collectstatic.

“Template does not exist” or template changes not showing

Stop all runserver processes and relaunch a single one — stale processes can serve a cached view (see port 8000 snippet above).

Migrations keep re-creating

python manage.py migrate --fake-initial

Reset everything (SQLite)

Remove-Item db.sqlite3  # or: rm db.sqlite3
python manage.py migrate
python manage.py loaddata fixtures/initial_data.json
